main.jsx â€” React browser ke sath kaise connect hoti hai (deep)

Yahan se React app DOM ke sath attach hoti hai. Example code aur har line ka matlab niche: 

// src/main.jsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App'

const root = ReactDOM.createRoot(document.getElementById('root'))
root.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
)
Line-by-line explanation (Roman Urdu)

import React â€” React library ko import kar rahe ho. JSX ko transpile karne ke liye zaroori.
import ReactDOM... â€” React ko browser DOM se connect karne ke liye. ReactDOM.createRoot modern API hai.
ReactDOM.createRoot(...) â€” Yeh ek `root` object banata jo tumhare #root div se linked hai.
root.render(<App/>) â€” Is se App component virtual DOM me render hota aur phir actual DOM me inject hota.

Roman: Main.jsx waiter hai jo chef ko bolta hai "is plate me mujhe dish serve kar" â€” phir dish (App) screen pe aa jati hai.

Mounting vs Hydration

Agar server-side rendering (SSR) use kar rahe ho toh hydrateRoot use hota hai. Normal client app me createRoot. 

// SSR hydation example (short)
import { hydrateRoot } from 'react-dom/client'
hydrateRoot(document.getElementById('root'), <App />)

Virtual DOM & Diffing

React DOM ko directly baar baar change karne ke bajaye ek JS object (virtual DOM) banata. Jab state change hoti hai, React purani aur naye virtual DOM ka farq (diff) nikalta, phir sirf required updates real DOM pe apply karta.

Simple Roman Urdu analogy

Socho tumare paas ek menu board hai (virtual DOM). Jab ek item price change ho to tum sirf usi item ko naya likh dete ho; puri board ko dobara paint nahi karte. Isi tara React sirf change wale nodes update karta â€” fast.

Components, Props, State

React component chhota piece hota hai jo UI banata. Parent se child ko data props se bhejte hain. Component apna internal data state rakhta hai jo useState se manage hota. 

// Example: src/App.jsx
import React, { useState } from 'react'

export default function App(){
  const [count, setCount] = useState(0)
  return (
    <div>
      <h1>Orders: {count}</h1>
      <button onClick={() => setCount(c => c + 1)}>Add</button>
    </div>
  )
}

Roman: Props: waiter ko chef ka note; State: table ke current order ka hal.

Hooks â€” useState, useEffect (aur custom hooks)

Hooks functions hain jo functional components me side-effects aur state handle karne dete hain. 

// useEffect example
import React, { useEffect, useState } from 'react'

function Clock(){
  const [time, setTime] = useState(new Date())
  useEffect(()=>{
    const id = setInterval(()=> setTime(new Date()), 1000)
    return ()=> clearInterval(id) // cleanup
  }, [])
  return <div>{time.toLocaleTimeString()}</div>
}
Roman: useEffect ka matlab

Jab component mount ho to effect chalana (componentDidMount). Agar dependency array [] hai to sirf ek baar chalega. Agar cleanup return kia hai to jab component unmount ho to cleanup chalta â€” memory leak se bachne ke liye.

Vite Dev Server & HMR (Hot Module Replacement)

Vite ek modern dev server hai jo modules ko fast serve karta. Har file change pe whole page reload nahi hoti â€” HMR sirf woh module update karta hai jisme change hua. 

// Common commands (terminal)
npm create vite@latest my-app --template react
cd my-app
npm install
npm run dev   # dev server (HMR)
npm run build # production build
npm run preview # preview build locally
Roman: Vite analogy

Vite kitchen me sirf usi dish ko update karta hai jo change hui â€” puri kitchen ko band nahi karta.

Build & Deploy

Production build me Vite tumhari app ko optimized files me bundle karta (minify, tree-shaking). Output dist/ folder hota hai jise Netlify, Vercel, GitHub Pages ya kisi static host pe deploy karte hain. 

npm run build
# dist/ ban jayega â€” isko host pe upload kar do
Tips (Roman)
  • Env variables: .env, .env.production me VITE_ prefix required.
  • Public folder: static assets yahan rakhna.
  • Bundle size check: analyzers use kar sakte ho.
